  • Link to the Base Plugin (zip)
  • This plugin enhances jQuery with a long range of extras, but most importantly it does this without keeping you out of the loop. The slider plugin does NOT create any wrapping boxes, nor does it append odd code to your existing code. Provide a wrapper for the slider and it's more than happy to use that! Same goes for the dialogue plugin. You hand it a DIV and it turns it into a dialogue. It doesn't take an existing DIV, creating less overhead.
    When this is said and done the Base Plugin has the option of adding elements to yours. Like an overview or buttons for control. This is entirely optional. Everything in this plugin can be custom styled.
  • Hold ctrl down and click on an element below to get more information and source code.
  • This library supports the GSAP TweenMax animation library from Greensock* and will use it instead of jquery automatically if imported.

    *I am in no way affiliated with Greensock, I do however really like the library; Check it out :)

  • Documentation:

    Main Method

    $(selector).rSlide({methods});

    rSlide will use a try/catch and use TweenMax from Greensock if applicable, if not it will revert to jQuery animations.

    Single Use Methods

    Each of these methods are stored in the data set and does not need to be re-set. You can access the data with $(element).data("slideData").method
    $(element).data("slideData").counter is especially usefull since it holds the current page number.

    • Gettables

      Any parameter that is set in rSlide() is saved in the elements data attribute. There are some internal variables that are saved as well:

      $(element).data("slideData").counter

      Returns the current slide index.

      $(element).data("slideData").childrenLength

      Gets the the amount of containers in the sliders using .length in-code. you may have to deduct 1 from this method to get a 0 indexed number.

    • Trim

      trim: true

      This method trims the content container to fit the containers exactly. It is sort of a useless ability, but may be useful on some rare cases. Please note that this action cannot be reverted via the plugin. If trim is retrieved from the data set, and has been set - it will just return true. If not it will return undefined or null.

    • endBack

      endBack: true

      This method makes the content slider go back to first slide when the user attempts to scroll past the last content container.

      endBackSpeed: int

      This method holds an integer that controls at which speed the scroller will go back at when using endBackSpeed.

    • overview

      overview: true

      This method adds an overview. Setting this to false will remove the overview

      overviewClass: "string"

      This method sets the class to the Ul controling the overview. Default is "overviewUl"

      overviewLiClass: "string"

      This method sets the class to the Li's controling the overview. Default is "overviewLI"

      overviewAddTo: jQuery object or string

      This method adds the overview to the object instead of the slider's parent. Default is undefined (parent)

    • arrows

      arrows: true

      This method adds an overview. Setting this to false will remove both arrows and unlink fading

      arrowsClass: "string"

      This method sets the class to the Ul controling the overview. Default is "overviewUl"

      arrowsLeftClass: "string"

      This method sets the class of the left button.

      arrowsRightClass: "string"

      This method sets the class of the right button.

      arrowAddTo: jQuery object or string

      This method adds the arrows to the object instead of the slider's parent. Default is undefined (parent)

      arrowsFade: true

      makes the appropriate arrow fade out when there's nothing more to slide to

    • includeMargin

      includeMargin: true

      This method makes the slider respect margins.

    • easing

      easing: string

      Changes the easing method from "swing" to whatever is defined in the string. Remember to use appropriate easings; Greensock uses Circ.easeIn and jQuery uses easeInCirc

    • speed

      speed: int

      Sets the speed in miliseconds at which the slider will slide.

    Methods

    • direction

      direction: "right"

      direction: "left"

      This method makes the slider slide in the desired direction

    • goTo

      goTo: int

      This method makes the slider go to a specific page - it is 0-indexed.

  • Documentation:

    Main Method

    $(selector).bauble({methods});

    The bauble will append an element to the parent. It can be called as many times as desired. The parent will be set as position: relative!

    Methods

    • align

      align: "top"

      align: "right"

      align: "bottom"

      align: "left"

      This sets the bauble alignment to whatever is chosen. Default is "bottom"

    • align2

      align2: "top"

      align2: "right"

      align2: "bottom"

      align2: "left"

      This sets the bauble alignment to whatever is chosen. This alignment is appended to align. This allows for combinations like: "bottom, left". It is not required.

    • x,y,z

      x: int

      y: int

      z: int

      x and y offsets the bauble after normal css values. z sets the z-index.

    • size

      size: int

      Sets the size of the bauble.

    • color

      color: hexadecimal string

      Sets the color of the bauble, usual CSS values here like: #ffffff.

    • flip

      flip: true

      Flips the bauble's local orientation.

    • permanent

      permanent: boolean

      If true the bauble will always be visible, if not it will only be visible on parent hover. default is true.

    • objId

      objId: string

      Gives the bauble an ID.

    • objClass

      objClass: string

      Gives the bauble a class.

  • Documentation:

    Main Method

    $(selector).createIcon({methods});

    The icon is appended to the parent. It can be called as many times as desired. The parent will be set as position: relative!

    Methods

    • title

      title: "top"

      This sets the title (name) of the icon and serves to determine filetype. Default is "file.misc"

    • imgDir

      imgDir: path string

      This sets the path to the folder containing the image-icons, default is "images/icon/".

    • customIcon

      customIcon: path string

      This overrides the normal filetype check and sets a custom icon image

    • objId

      objId: string

      Gives the icon an ID.

    • objClass

      objClass: string

      Gives the icon a class.
